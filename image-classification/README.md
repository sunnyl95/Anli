# 图像分类案例

## 案例说明
1.对`飞机`、`太阳`、`海星`、`花`四类图像进行分类<p>
2.目的是作为`数据交易沙箱`平台`非结构化数据`测试的案例<p>
3.测试的内容包括：<p>
`--`上传非结构化数据源<p>
`--`非结构化数据源抽样<p>
`--`样本数据授权数据分析师<p>
`--`创建notebook实验<p>
`--`调试环境运行<p>
`--`全量运行:评估指标、训练过程图、生成版本、版本比对<p>
`--`git版本管理<p>



## 目录结构描述
```
├── Readme.md                   // 项目说明 
├── image_train                    // 用于训练的图像数据集               
│   ├── flower               // 花的图片集 
	├── plane               // 飞机的图片集 
	├── starfish               // 海星的图片集 
	├── sun               // 太阳的图片集
├── image_test                    // 用于测试的图像数据集              
│   ├── flower               // 花的图片集 
	├── plane               // 飞机的图片集 
	├── starfish               // 海星的图片集
	├── sun               // 太阳的图片集
├── train.ipynb              // 训练分类器 （pytorch）                

```


## run
1.将image_train和image_test压缩成zip包，分别上传全量数据源<p>
2.按照0.5比例抽样，并上送至调试环境<p>
3.配置策略，将抽样数据授权给数据分析师<p>
4.创建notebook实验，选择上述两个抽样数据集<p>
5.将`train.ipynb`上传至notebook实验界面<p>
6.替换`train.ipynb`里的`_INPUT1`变量里的`uri`为你抽样训练集路径；同样替换`train.ipynb`里的`_INPUT2`变量里的`uri`为你抽样测试集路径，点击保存<p>
7.在调试环境中运行`train.ipynb`，如果没问题，在在当前目录下生成三个文件夹――"traindata"、"testdata"和"models"。"traindata"和"testdata"是从S3里下载下来的抽样图片数据，models文件夹下的`mycnn.pth`表示保存的模型。<p>
8.如果步骤7没问题，即可点击【全量运行】在全量数据上完成训练<p>
9.在【全量运行记录】中可看到全量运行的结果，点击【结果申报】，即可申请全量数据训练的模型。<p>
10.运行环境管理员结合敏感数据检测引擎，审核申请的内容，无问题，数据分析师即可在【全量运行记录】界面下载模型<p>
11.多次全量运行可以在【版本对比】里选择任意两个版本结果进行比对<p>
